◆ open_file_callback

typedef TftpdOperationResult(* open_file_callback) (const TftpdSectionHandlerPtr section_handler, FILE **fd, char *filename, char *mode, size_t *bufferSize, void *context)

Open file callback.

This callback is called when the server needs to open a file, so a file pointer must be opened for the filename with the given mode.

If you want to use the default open file function, don't register this callback. The server will open the file based on the file name received.

If you open the file, you must close it. Use the close_file_callback for that.

Parameters
[in]section_handlerthe section handler.
[out]fdthe file pointer to open.
[in]filenamethe name of the file to open.
[in]modethe mode to open the file in.
[out]bufferSizeif a fmemopen is used, this parameter must be set to the size of the buffer. If a normal fopen is used, this parameter must be set to 0.
[in]contextthe user context.
Returns
TFTPD_OK if success.
TFTPD_ERROR otherwise.

◆ set_server_timeout()

TftpdOperationResult set_server_timeout ( const TftpdHandlerPtr  handler,
const int  timeout 
)

Set timeout.

This is the time the server will wait for a client to send a request. If the client doesn't send a request within this time, the server will close the connection. The default is 300 seconds. If you want to disable the timeout, set the timeout to 0.

Parameters
[in]handlerthe pointer to the tftpd handler.
[in]timeoutthe timeout in seconds.
Returns
TFTPD_OK if success.
TFTPD_ERROR otherwise.

TftpdOperationResult stop_listening ( const TftpdHandlerPtr  handler)

Request the TFTPD to stop listening.

The success return of this function doesn't mean the server has stopped, it just means that the server will stop listening when it is ready. When the server stops listening, the start_listening() function will return.
